# Architectural Motif Library

Use this reference after observing the supplied image. Choose the smallest motif that explains the interface hierarchy.

| Observed feature | UI translation | Good use | Avoid |
| --- | --- | --- | --- |
| Layered upturned eaves | A two- or three-step section silhouette | Page corner, section divider, level marker | Roofs on every card or literal temple icons |

| Lantern | One small warm accent | Current item or live status, with a text label | Repeated red dots as the only state indicator |
| Ink/paper print | Hard outline, flat fill, subtle grain | Poster panels and lesson specimens | Glassmorphism, soft cream SaaS cards, fake brush strokes |

## Reference-specific observation

The supplied MoZhi reference showed a misty, dark-timber gate or bridge with stacked rooflines, small red lantern accents, surrounding foliage, and a reflection in still water. Translate its hierarchy as dark structural framing, quiet negative space, one red accent, and a grounded reflection. Do not reproduce the building silhouette or imply a specific site, era, or regional origin without source evidence.

## Originality test

Before implementation, answer all four:

1. Which visible architectural principle was extracted?
2. How was its geometry or proportion transformed?
3. What interface function does it perform?
4. What was intentionally omitted to avoid copying or cultural shorthand?

If a motif has no interface function, remove it.
